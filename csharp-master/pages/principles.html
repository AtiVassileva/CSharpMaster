OOП в C#: Наследяване

Наследяването е основен принцип в обектно-ориентираното програмиране (OOP). То позволява на един клас (наследник) да наследи функционалностите на друг клас (родител).

Пример за наследяване в C#

Нека разгледаме как един клас Dog може да наследи от базовия клас Animal:

public class Animal 
{
    public void MakeSound() 
    {
        Console.WriteLine("Животното издава звук!");
    }
}

public class Dog : Animal 
{
    public void Bark() 
    {
        Console.WriteLine("Кучето лае!");
    }
}

public class Program 
{
    public static void Main() 
    {
        Dog myDog = new Dog();
        myDog.MakeSound(); // Наследен метод
        myDog.Bark(); // Собствен метод
    }
}

Илюстрация

В този пример Dog и Cat наследяват класа Animal, както и метода MakeSound() от Animal.Освен това, те добавят и свои методи Bark() и Meow().

OOП в C#: Енкапсулация

Енкапсулацията е принцип в обектно-ориентираното програмиране, който ограничава достъпа до данните в един клас, като ги защитава от директна модификация.

Пример за енкапсулация в C#

Нека разгледаме как можем да скрием данни в един клас и да имаме контрол над достъпа:

            public class BankAccount 
            {
                private double balance; // Частна променлива
            
                public void Deposit(double amount) 
                {
                    if (amount > 0) 
                    {
                        balance += amount;
                    }
                }
            
                public double GetBalance() 
                {
                    return balance;
                }
            }
            
            public class Program 
            {
                public static void Main() 
                {
                    BankAccount account = new BankAccount();
                    account.Deposit(100);
                    Console.WriteLine("Баланс: " + account.GetBalance());
                }
            }

В този пример променливата balance е скрита и достъпна само чрез методите Deposit() и GetBalance().


Илюстрация

На снимката енкапсулацията е представена чрез капсула, която символизира клас, съдържащ вътре променливи (червени топчета) и методи (зелени топчета). Това подчертава идеята, че данните и функциите са капсулирани заедно в класа, осигурявайки контрол над достъпа.

OOП в C#: Полиморфизъм

Полиморфизмът е един от основните принципи на обектно-ориентираното програмиране, който позволява на един интерфейс да бъде използван за различни типове обекти.

Пример за полиморфизъм в C#

Нека разгледаме как различни класове могат да имат еднакви методи, но с различна реализация:

            public class Animal 
            {
                public virtual void MakeSound() 
                {
                    Console.WriteLine("Животното издава звук.");
                }
            }
            
            public class Dog : Animal 
            {
                public override void MakeSound() 
                {
                    Console.WriteLine("Кучето лае.");
                }
            }
            
            public class Cat : Animal 
            {
                public override void MakeSound() 
                {
                    Console.WriteLine("Котката мяука.");
                }
            }
            
            public class Program 
            {
                public static void Main() 
                {
                    Animal myDog = new Dog();
                    Animal myCat = new Cat();
                    
                    myDog.MakeSound(); // Кучето лае.
                    myCat.MakeSound(); // Котката мяука.
                }
            }

В този пример методът MakeSound() е дефиниран в базовия клас, но е презаписан чрез (override) в наследените класове, което позволява различно поведение.


Илюстрация

Картинката илюстрира концепцията за полиморфизъм чрез един и същ герой, който изпълнява различни роли – например пазарува, работи на компютър, говори по телефона, представя презентация, грижи се за дете и дори е в супергеройска поза. Това показва, че един обект (човекът) може да има различни форми и поведения в зависимост от контекста, което е основен принцип в обектно-ориентираното програмиране.

OOП в C#: Абстракция

Абстракцията е принцип в обектно-ориентираното програмиране, който позволява да се скрият детайлите на имплементацията и да се предостави само необходимата функционалност.

Пример за абстракция в C#

В следващия пример ще използваме абстрактен клас, за да дефинираме обща структура за всички превозни средства:

            public abstract class Vehicle 
            {
                public abstract void StartEngine(); // Абстрактен метод
            }
            
            public class Car : Vehicle 
            {
                public override void StartEngine() 
                {
                    Console.WriteLine("Колата стартира двигателя.");
                }
            }
            
            public class Motorcycle : Vehicle 
            {
                public override void StartEngine() 
                {
                    Console.WriteLine("Мотоциклетът стартира двигателя.");
                }
            }
            
            public class Program 
            {
                public static void Main() 
                {
                    Vehicle myCar = new Car();
                    Vehicle myMotorcycle = new Motorcycle();
                    
                    myCar.StartEngine(); // Колата стартира двигателя.
                    myMotorcycle.StartEngine(); // Мотоциклетът стартира двигателя.
                }
            }

В този пример класът Vehicle дефинира абстрактен метод StartEngine(), който се имплементира различно в неговите наследници.


Илюстрация

Картинката илюстрира концепцията за абстракция чрез примери с банкомат (ATM) и кафе машина. Потребителите използват тези устройства, без да знаят вътрешната им реализация – те виждат само необходимите функции, като теглене на пари или приготвяне на кафе. Това съответства на принципа на абстракция в програмирането, където сложността се скрива, а се предоставя само съществената функционалност.
